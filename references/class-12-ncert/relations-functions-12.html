    Relations and Functions (Class 12, Chapter 1)

    A written tutorial covering the NCERT Class 12 chapter on relations and functions: types of relations (reflexive, symmetric, transitive, equivalence), equivalence classes, types of functions (one-one, onto, bijective), composition of functions, and invertibility. The chapter assumes Class 11 fluency with sets, ordered pairs, and the basic definitions of relation and function.

    1. Quick recap from Class 11

    You already met these:

    • Set. A well-defined collection of objects: A = {a, e, i, o, u} is the set of vowels in English.

    • Cartesian product. For two sets A and B, A × B = { (a, b) : a ∈ A, b ∈ B }. If A = {1, 2} and B = {3, 4, 5}, then

      A × B = {(1,3), (1,4), (1,5), (2,3), (2,4), (2,5)}

    • Relation from A to B. Any subset of A × B. A relation just picks out some of the ordered pairs. For A and B above, the relation R = {(1,3), (2,4), (3,5)} (or its set-builder form {(x, y) : y = x + 2}) is one of many possible relations.

    • Function from A to B. A relation in which every element of A is paired with exactly one element of B. Notation f : A → B and f(a) = b.

    Class 12 picks up by asking: among all the relations on a single set, which have nice structural properties? Among all the functions between two sets, which can be reversed?

    2. Types of relations on a set

    For this whole section, fix a non-empty set A. A relation on A is a subset of A × A.

    Two extreme examples:

    Relation Definition Example for A = {1, 2, 3}
    Empty relation R = ∅ (no element of A related to any element) {}
    Universal relation R = A × A (every element related to every element) {(1,1), (1,2), (1,3), (2,1), … , (3,3)} (9 pairs)

    Everything in between is some intermediate relation. The three structural properties that organise them are reflexive, symmetric, and transitive.

    2.1 Reflexive

    R is reflexive if (a, a) ∈ R for every a ∈ A. Every element is related to itself.

    Examples.

    • “is equal to” on any set — reflexive (everything equals itself).
    • “is a brother of” on the set of people — not reflexive (you are not your own brother).
    • On A = {1, 2, 3}, R = {(1,1), (2,2), (3,3), (1,2)} is reflexive; R = {(1,1), (2,2), (1,2)} is not ((3, 3) is missing).

    2.2 Symmetric

    R is symmetric if (a, b) ∈ R implies (b, a) ∈ R for all a, b ∈ A.

    Examples.

    • “is a sibling of” — symmetric.
    • “is taller than” — not symmetric.
    • On A = {1, 2, 3}, R = {(1, 2), (2, 1)} is symmetric. R = {(1, 2)} is not.

    2.3 Transitive

    R is transitive if (a, b) ∈ R and (b, c) ∈ R imply (a, c) ∈ R for all a, b, c ∈ A.

    Examples.

    • “is equal to”, “is a multiple of”, “is less than” — all transitive.
    • “is the mother of” — not transitive (mother’s mother is grandmother, not mother).

    2.4 Equivalence relation

    R is an equivalence relation if it is reflexive and symmetric and transitive. All three.

    The single most important example: equality. The second most important: congruence modulo n on the integers. The third most: “lies on the same line through the origin” on ℝ² ∖ {(0,0)}.

    3. Worked examples on types of relations

    3.1 The “set of books with the same number of pages”

    A = all books in a library. Define R = { (x, y) : x and y have the same number of pages }. Show R is an equivalence relation.

    • Reflexive. Any book x has the same number of pages as itself, so (x, x) ∈ R.
    • Symmetric. If (x, y) ∈ R, then x and y have the same number of pages, so y and x do too, so (y, x) ∈ R.
    • Transitive. If (x, y) ∈ R and (y, z) ∈ R, then x, y have the same number of pages and y, z have the same number of pages. So x, z have the same number of pages, hence (x, z) ∈ R.

    All three properties hold, so R is an equivalence relation.

    3.2 The “difference is divisible by 4” relation

    A = {x ∈ ℤ : 0 ≤ x ≤ 12} = {0, 1, 2, …, 12}. Define R = { (a, b) : |a − b| is a multiple of 4 }. Show R is an equivalence relation.

    • Reflexive. |a − a| = 0, which is a multiple of 4 (since 0 = 4·0). So (a, a) ∈ R.
    • Symmetric. If |a − b| = 4k, then |b − a| = |−(a − b)| = |a − b| = 4k too. Symmetric.
    • Transitive. If |a − b| = 4k₁ and |b − c| = 4k₂, both a − b and b − c are integer multiples of 4. Their sum (a − b) + (b − c) = a − c is also a multiple of 4, so |a − c| is a multiple of 4. Hence (a, c) ∈ R.

    So R is an equivalence relation.

    3.3 The “differ by an even number” relation

    A = {1, 2, 3, 4, 5} and R = { (a, b) : |a − b| is even }. Equivalence relation? Yes:

    • Reflexive: |a − a| = 0, even.
    • Symmetric: |a − b| = |b − a|.
    • Transitive: sum of two even numbers is even, so if |a−b| and |b−c| are even, so is |a−c|.

    Worth noting: R relates the odd elements {1, 3, 5} to each other (all pairwise differences are even) and the even elements {2, 4} to each other, but no odd is related to any even. The set A splits into the two “blocks” {1, 3, 5} and {2, 4}. These blocks are equivalence classes — the next concept.

    4. Equivalence classes

    Whenever R is an equivalence relation on A, the elements of A split into disjoint blocks called equivalence classes. The class of a is

    [a] = { x ∈ A : (a, x) ∈ R }

    Two key theorems:

    1. Every element of A lies in exactly one equivalence class.
    2. Two classes [a] and [b] are either identical (if (a, b) ∈ R) or disjoint (if not).

    So an equivalence relation on A is the same data as a partition of A into non-overlapping blocks.

    Example. In §3.3 the equivalence classes are {1, 3, 5} and {2, 4}.

    Example. In §3.2 the equivalence classes of “differ by a multiple of 4” on {0, 1, …, 12} are:

    • {0, 4, 8, 12} — multiples of 4
    • {1, 5, 9} — remainder 1 mod 4
    • {2, 6, 10} — remainder 2 mod 4
    • {3, 7, 11} — remainder 3 mod 4

    5. Types of functions

    Now switch perspectives. A function f : A → B assigns to each a ∈ A exactly one b ∈ B, called the image f(a). The set A is the domain, B the codomain, and f(A) = { f(a) : a ∈ A } ⊆ B is the range (or image).

    The two structural properties that organise functions are injectivity and surjectivity.

    5.1 One-one (injective)

    f is one-one (or injective) if different inputs give different outputs. Formally:

    For all x₁, x₂ ∈ A: f(x₁) = f(x₂)  ⇒  x₁ = x₂

    Equivalently, x₁ ≠ x₂ ⇒ f(x₁) ≠ f(x₂).

    5.2 Onto (surjective)

    f is onto (or surjective) if every element of the codomain has at least one pre-image. Formally:

    For all y ∈ B, there exists x ∈ A with f(x) = y

    Equivalently, the range equals the codomain: f(A) = B.

    5.3 Bijective (one-one and onto)

    A function that is both one-one and onto is called bijective. Bijective functions are exactly those that can be reversed (see §7).

    5.4 Examples

    f : A → B one-one? onto?
    f : ℝ → ℝ, f(x) = x yes yes
    f : ℝ → ℝ, f(x) = 2x + 1 yes yes
    f : ℝ → ℝ, f(x) = x² no (f(2) = f(−2) = 4) no (no negative y is hit)
    f : ℕ → ℕ, f(x) = x² yes (different naturals give different squares) no (2, 3, 5, … not hit)
    f : ℝ → ℝ, f(x) = 1/x (with domain ℝ \ {0}, range ℝ \ {0}) yes yes (range = codomain)
    f : ℤ → ℤ, f(x) = x² no no

    5.5 How to show a function is one-one (standard procedure)

    Set f(x₁) = f(x₂), manipulate, and conclude x₁ = x₂.

    Example. f : ℝ → ℝ, f(x) = 2x + 3.

    2x₁ + 3 = 2x₂ + 3
2x₁ = 2x₂
x₁ = x₂        ✓ one-one

    Example. f : ℕ → ℕ, f(x) = x².

    x₁² = x₂²
x₁ = ±x₂

    Since x₁, x₂ ∈ ℕ, both are positive, so x₁ = x₂. One-one on ℕ. (Note: on ℤ this argument fails — ±x₂ includes a negative possibility, and f(2) = f(−2).)

    5.6 How to show a function is onto (standard procedure)

    Pick an arbitrary y in the codomain. Solve f(x) = y for x and check that x lies in the domain.

    Example. f : ℝ → ℝ, f(x) = 2x + 3. Solve 2x + 3 = y ⇒ x = (y − 3)/2. For every real y, (y − 3)/2 is a real number, so a pre-image always exists. Onto.

    Example. f : ℝ \ {0} → ℝ \ {0}, f(x) = 1/x. Solve 1/x = y ⇒ x = 1/y. For every non-zero y, 1/y is a non-zero real, so a pre-image always exists. Onto.

    Counterexample. f : ℕ → ℕ, f(x) = x². Try y = 2. We’d need x² = 2, but no natural number squares to 2. So 2 has no pre-image. Not onto.

    6. Counting one-one and onto functions

    For finite sets A (size m) and B (size n):

    Question Answer
    How many functions f : A → B? nᵐ
    How many one-one functions f : A → B? n · (n−1) · (n−2) · … · (n−m+1) if m ≤ n, else 0
    How many onto functions f : A → B when m = n? n!
    How many bijections f : A → A? n!

    The n! count for onto functions from a set to itself comes from the same argument as for bijections: the first input has n choices for its image, the second has n − 1 remaining choices (must use a different image to stay onto / one-one), and so on.

    7. Composition of functions

    Given f : A → B and g : B → C, the composition g ∘ f : A → C is defined by

    (g ∘ f)(x) = g(f(x))

    You first apply f, then apply g to the result.

    7.1 Examples

    f(x) = x + 2, g(x) = x². Then

    • (g ∘ f)(x) = g(f(x)) = g(x + 2) = (x + 2)² = x² + 4x + 4
    • (f ∘ g)(x) = f(g(x)) = f(x²) = x² + 2

    In general g ∘ f ≠ f ∘ g. Composition is not commutative.

    7.2 Properties

    • Associative. h ∘ (g ∘ f) = (h ∘ g) ∘ f.
    • Identity composition. If id_A and id_B are identity functions, then f ∘ id_A = f and id_B ∘ f = f.
    • Composition of one-one functions is one-one. Composition of onto functions is onto. Composition of bijections is a bijection.

    7.3 Worked example

    f : ℝ → ℝ, f(x) = 2x − 1. g : ℝ → ℝ, g(x) = (x + 1)/2. Compute g ∘ f and f ∘ g:

    (g ∘ f)(x) = g(2x − 1) = ((2x − 1) + 1) / 2 = 2x/2 = x
(f ∘ g)(x) = f((x + 1)/2) = 2 · (x + 1)/2 − 1 = (x + 1) − 1 = x

    Both compositions give the identity. This is a clue that f and g are inverses of each other — see next section.

    8. Invertible functions

    A function f : A → B is invertible if there exists g : B → A such that

    g ∘ f = id_A   and   f ∘ g = id_B

    The function g is then called the inverse of f, written f⁻¹.

    8.1 Existence criterion

    A function f : A → B is invertible iff it is bijective.

    Why: to invert f, you need to send each b ∈ B back to a unique a ∈ A. The “back” needs at least one candidate (onto) and at most one candidate (one-one).

    8.2 How to find the inverse

    1. Write y = f(x).
    2. Solve for x in terms of y.
    3. Swap x and y (or just rename) to get f⁻¹(x).

    Example. f : ℝ → ℝ, f(x) = 2x − 1.

    • y = 2x − 1 ⇒ x = (y + 1)/2.
    • So f⁻¹(y) = (y + 1)/2, or equivalently f⁻¹(x) = (x + 1)/2.

    Compare to §7.3 — exactly the g we computed.

    8.3 Inverse-of-inverse and inverse-of-composition

    • (f⁻¹)⁻¹ = f.
    • (g ∘ f)⁻¹ = f⁻¹ ∘ g⁻¹ (mind the order — inverses compose in reverse).

    8.4 Worked example with restricted domain

    Sometimes a function isn’t invertible on its natural domain but becomes invertible if we shrink the domain. For example f(x) = x² on ℝ is not one-one (and not onto when codomain is ℝ). But f : [0, ∞) → [0, ∞) is bijective, with inverse f⁻¹(x) = √x. This restricted-domain construction is exactly how √ and the inverse trig functions in the next chapter are defined.

    9. Common pitfalls

    • Reflexivity needs every element to relate to itself. R = {(1, 1), (2, 2)} on A = {1, 2, 3} is not reflexive — (3, 3) is missing.
    • Symmetry needs both directions. A relation can contain (1, 2) without (2, 1). Always check pairs.
    • Transitivity is the trickiest of the three. You must check that for every combination of (a, b) and (b, c) already in R, (a, c) is also in R.
    • Two-step transitivity from a chain. Don’t confuse the property with the closure. The relation {(1, 2), (2, 3)} is not transitive because (1, 3) is missing. Adding (1, 3) gives {(1, 2), (2, 3), (1, 3)}, which is transitive.
    • Equivalence relation ⟹ partition. If you’re asked for equivalence classes, every element must be placed in exactly one class.
    • Range vs codomain. Onto means the range equals the codomain, not that the range is some other big set. f : ℝ → ℝ, f(x) = x² is not onto because the codomain ℝ contains negatives but the range [0, ∞) does not.
    • Powers of i² = −1 style errors with x² = y. When solving x² = y for x, both +√y and −√y are candidates. On ℕ only the positive root is in the domain; on ℤ both are; on ℝ both are (if y ≥ 0).
    • Composition order. g ∘ f means first f, then g. Easy to flip when writing on paper.
    • Inverse exists only if bijective. Don’t apply f⁻¹ to a function that isn’t one-one or isn’t onto.

    10. Quick reference card

    RELATIONS ON A

reflexive          (a, a) ∈ R  for all a
symmetric          (a, b) ∈ R  ⇒  (b, a) ∈ R
transitive         (a, b), (b, c) ∈ R  ⇒  (a, c) ∈ R
equivalence        reflexive AND symmetric AND transitive

equivalence class  [a] = { x : (a, x) ∈ R }
partition          equivalence classes partition A

FUNCTIONS f : A → B

one-one (injective)    f(x₁) = f(x₂) ⇒ x₁ = x₂
onto (surjective)       for every y ∈ B, ∃ x ∈ A: f(x) = y
bijective              one-one AND onto

composition            (g ∘ f)(x) = g(f(x))
identity               id_A : A → A,  id_A(x) = x
invertibility           f is invertible ⟺ f is bijective
                        (g ∘ f)⁻¹ = f⁻¹ ∘ g⁻¹

COUNTING (|A| = m, |B| = n)
  total functions       nᵐ
  one-one functions     n·(n−1)·…·(n−m+1)  if m ≤ n
  onto functions A→A    n!
  bijections A→A         n!

    Procedure to test a relation:

    Pick R ⊆ A × A.
  Reflexive? Check (a, a) ∈ R for every a.
  Symmetric? Check (a, b) ∈ R ⇒ (b, a) ∈ R.
  Transitive? Check (a, b), (b, c) ∈ R ⇒ (a, c) ∈ R.
  All three? Equivalence.

    Procedure to test/find function properties:

    One-one:   f(x₁) = f(x₂)  →  solve  →  must force x₁ = x₂.
Onto:      pick arbitrary y in codomain  →  solve f(x) = y for x
           →  check x is in the domain.
Inverse:   y = f(x)  →  solve for x in terms of y  →  rename.

    11. Looking ahead

    • Inverse trigonometric functions (Chapter 2) are an application of invertibility: trig functions on restricted domains become bijective and the inverses get standard names (sin⁻¹, cos⁻¹, etc.).
    • Binary operations (often in the same chapter in older syllabi) are functions * : A × A → A. The properties commutative, associative, identity-element-existence, and inverse-existence give group structure. The integers under addition form a group; the real numbers under multiplication do not (0 has no inverse).
    • Linear algebra. Matrices give the most fertile source of one-one / onto / bijective questions — a square matrix is invertible iff the corresponding linear map is bijective iff the determinant is non-zero.
    • Modular arithmetic. “Congruent mod n” on ℤ is the canonical equivalence relation, with n classes [0], [1], …, [n − 1]. The set of classes itself supports its own arithmetic.

    The single habit to internalise: every time you’re asked to verify a relation has a property, write down what the property requires, then check it pair by pair. Every time you’re asked to show a function is one-one or onto, write down the definition of one-one or onto and march straight through it. The structure of these proofs is always the same — only the algebra in the middle changes.
