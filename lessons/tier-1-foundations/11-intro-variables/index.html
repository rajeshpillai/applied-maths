11 · Variables

When the number isn’t yet known, give it a name and reason about it

Pre-flight check

  1. Have you done Lesson 4 — Add & Subtract and Lesson 5 — Multiplication as Area?
  2. In Lesson 4 (P2), you found the missing number in \ ? + 36 = 100. Did that feel like a new kind of computation — solving for the unknown — rather than just doing arithmetic?
  3. Have you used a spreadsheet, where one cell’s value depends on others? That dependence is exactly the variable idea.

Touch

A simple number-machine. The machine takes an input x, multiplies it by 2, then adds 5. Drag x. Watch the output.

The letter x is a variable — a placeholder for any number you care to feed in. The expression 2x + 5 describes a recipe. The output y depends on x via that recipe.

Wonder

  1. If x = 0 what is y? If x = -3? If x = 100?
  2. The recipe 2x + 5 is one expression. Could you describe the same recipe with different letters? Would it still be the same?
  3. If you knew the output y = 17, could you work backwards to the input x?

Discover

Problem 1

Compute y = 2x + 5 for x = 0, 1, 2, 3, -2.

Full solution

y(0) = 5, \ y(1) = 7, \ y(2) = 9, \ y(3) = 11, \ y(-2) = 1.

Notice the pattern: as x goes up by 1, y goes up by 2. The “2” in 2x + 5 controls the rate of change; the “5” sets the value when x = 0.

You have just met your first linear function. Tier 2 will explore these in depth — they are the spine of much of applied mathematics.

Problem 2

A taxi charges Rs. 30 as a base fare, plus Rs. 15 per kilometre. Write an expression for the total cost C if you travel d kilometres.

Hint 1 — a nudge Base fare is fixed. Per-km charge depends on distance.
Full solution

C = 30 + 15 \times d, \quad \text{or simply} \quad C = 30 + 15d.

(Recall from Lesson 5 Stretch S2 that we drop the × sign in algebraic expressions because × looks like the letter x.)

You have written your first algebraic formula — an expression involving a variable (here, d) that captures a real-world rule. For d = 8 km: C = 30 + 15 \times 8 = 30 + 120 = 150 rupees.

Problem 3

Solve for the unknown: x + 7 = 12.

What is x?

Hint 1 — a nudge What plus 7 gives 12? Use the inverse operation (Lesson 4 P2).
Full solution

x = 5. Method: subtract 7 from both sides of the equation. x + 7 - 7 = 12 - 7 \quad \Longrightarrow \quad x = 5.

The “do the same thing to both sides” idea is the fundamental move of algebra. The equation is a balance scale; whatever you do to the left you must do to the right, or the balance breaks.

Problem 4

Solve for x: 3x = 21.

Hint 1 — a nudge Inverse of multiplication is division. Divide both sides by 3.
Full solution

x = 21 \div 3 = 7. Check: 3 \times 7 = 21 ✓.

To “isolate” a variable, undo the operations one at a time, using inverses, while keeping both sides equal.

Problem 5

Verify the distributive law by substituting numbers. The law says: for any a, b, c, a \times (b + c) = a \times b + a \times c.

Try a = 3, b = 4, c = 5. Compute both sides.

Full solution

Left: 3 \times (4 + 5) = 3 \times 9 = 27. Right: 3 \times 4 + 3 \times 5 = 12 + 15 = 27. ✓ Both equal 27.

Try with any other numbers — they will always agree. That is the meaning of “law”: a statement true for all values of the variables.

Picture (recall Lesson 5 Discover P4): a 3 \times 9 rectangle is the same as a 3 \times 4 rectangle and a 3 \times 5 rectangle placed side by side. The distributive law is the rectangle-splitting picture put into symbols.

This is the power of variables: one symbolic statement captures infinitely many numerical facts.

The French mathematician François Viète (1540–1603) was the first to use letters for both knowns and unknowns — vowels for unknowns, consonants for parameters — in his In artem analyticem isagoge (1591). Before Viète, “algebra” was done in words and ad-hoc abbreviations. The modern x, y, z for unknowns was a tweak introduced by Descartes in La Géométrie (1637); the letters stuck. The word algebra itself comes from al-Khwārizmī’s 9th-century al-jabr — “reunion of broken parts” — the move you make when you add the same thing to both sides.

A variable is a promise about a number, not the number itself. Once you treat it like a real number — adding, multiplying, doing the same thing to both sides — the unknown reveals itself.

Name

A variable is a letter (usually x, y, n, t, \ldots) that stands in for a number whose value is either unknown, or allowed to vary across some range.

An algebraic expression is a recipe built from numbers, variables, and operations: 2x + 5, 3x - 7, x^2, \frac{a + b}{2}.

An equation is a statement that two expressions are equal: x + 7 = 12. To solve an equation is to find the value(s) of the variable that make it true.

The fundamental move: whatever you do to one side of an equation, do to the other. (Add the same number, subtract the same number, multiply by the same non-zero number, etc.) This preserves equality.

A letter for a number we do not know; A recipe written once, used many times; Both sides equal — keep the balance, and the balance reveals the unknown.

Apply

Worked example

Solve for x: 2x + 5 = 17.

A competent solver works one inverse at a time:

  1. Subtract 5 from both sides to peel off the “+5”: 2x + 5 - 5 = 17 - 5 \quad \Longrightarrow \quad 2x = 12.
  2. Divide both sides by 2 to peel off the “× 2”: \frac{2x}{2} = \frac{12}{2} \quad \Longrightarrow \quad x = 6.
  3. Check: 2 \times 6 + 5 = 12 + 5 = 17 ✓.

Answer: x = 6. Always check by substituting your answer back into the original equation. It catches arithmetic slips immediately.

Practice — Warm

W1. Evaluate 3x - 4 for x = 0, 2, 5, -1.

Full solution -4, \ 2, \ 11, \ -7.

W2. Solve: \ x + 9 = 15. Then: \ y - 4 = 10.

Full solution x = 6 (subtract 9 from both sides). y = 14 (add 4 to both sides).

W3. Solve: \ 4x = 32. Then: \ \dfrac{y}{5} = 6.

Full solution x = 8 (divide both sides by 4). y = 30 (multiply both sides by 5).

Practice — Working

P1. Solve: \ 3x + 7 = 25.

Hint 1 — a nudge Subtract 7 first; then divide by 3.
Full solution 3x = 18, then x = 6. Check: 3 \times 6 + 7 = 25 ✓.

P2. A rectangular garden has length L metres and width 6 metres. Its area is 42 square metres. Find L.

Hint 1 — a nudge Area = length × width. Set up an equation.
Full solution 6L = 42, so L = 7 metres. The variable lets us write one equation that captures all possible problems of this shape — change the numbers and the same algebra still works.

P3. A taxi from Discover P2 charges C = 30 + 15d. The fare for a ride was Rs. 195. How far did the trip go?

Hint 1 — a nudge Set C = 195 and solve for d.
Full solution

30 + 15d = 195. Subtract 30: 15d = 165. Divide by 15: d = 11 km.

This is the power of equations: one formula solves both forward (given distance, find cost) and backward (given cost, find distance) problems.

Practice — Stretch

S1. The distributive law (Discover P5) lets us expand products of expressions. Compute (x + 3)(x + 5) in expanded form.

Hint 1 — a nudge Picture: a rectangle with one side (x + 3) and the other (x + 5). Split into 4 sub-rectangles.
Full solution

The rectangle splits into four pieces: x \cdot x, x \cdot 5, 3 \cdot x, 3 \cdot 5. Areas: x^2, 5x, 3x, 15.

(x + 3)(x + 5) = x^2 + 5x + 3x + 15 = x^2 + 8x + 15.

You just multiplied your first polynomial. Algebra II in Tier 3 will revisit this thoroughly. The rectangle picture from Lesson 5 is once again the engine behind everything.

S2. A neural network’s prediction for one input is computed as \hat{y} = w_1 x_1 + w_2 x_2 + b, where x_1, x_2 are the input values, w_1, w_2 are the weights, and b is the bias. Why are the variables grouped this way? What real problem does the algebra solve?

Hint 1 — a nudge Compare the structure to the taxi formula C = 30 + 15d.
Full solution

The neural network formula is exactly the taxi formula — generalised. b plays the role of the base fare; each weight w_i plays the role of “rate per unit of x_i”. The whole machinery of deep learning is built on stacking many of these linear formulas, with non-linear “squashing” functions in between.

When the network trains, it solves equations — but in reverse: given many input/output examples, it adjusts the w’s and b’s to make \hat{y} match the true y as closely as possible. The “fundamental move” of algebra (do the same thing to both sides) becomes the “fundamental move” of training (adjust the weights using the gradient).

You have just met the deepest connection between algebra and AI: a neural network is a (very) large algebraic expression, and learning is solving for its variables.

Common confusions

  1. “x is a fixed number.” Sometimes (when solving an equation, the solution x is fixed). But variables are also used to mean any number (when stating laws like the distributive law). Context picks.
  2. “You can do anything to one side of an equation.” No — only if you do it to the other side too. Otherwise, the balance breaks and the equation no longer holds.
  3. “Algebra is just arithmetic with letters.” Almost true — but the letters let you express general facts and infinitely many computations at once. That generalisation is what makes algebra powerful.

Takeaways

If you remember nothing else from this lesson, remember these:

  1. A variable is a placeholder for a number — sometimes one specific unknown, sometimes any number in a range.
  2. An equation is a balance: do the same thing to both sides and the balance holds — that one move solves almost every Tier 1 equation.
  3. One algebraic expression like 2x + 5 stands for infinitely many arithmetic facts at once — that compression is why algebra runs the modern world.

Where this matters

AI / Machine Learning. A neural network is, mathematically, a giant algebraic expression in (sometimes billions of) variables. Training is the process of finding the values of those variables that minimise a loss function. Stretch S2 makes the connection explicit.

Encryption. Modern cryptography is full of equations of the form “given g^x \bmod p, find x” (the discrete logarithm problem) — its hardness is what protects your messages. Variables and equations are the language in which these problems are stated.

Mathematical research. Variables and the equational style are the language of mathematics. From abstract algebra to differential equations to category theory, every advanced subject extends what you have just begun.

Everywhere else. Spreadsheets, programming (every variable in code), recipe scaling, financial planning, scientific modelling, engineering design — the moment you label something with a name and reason about it abstractly, you are doing algebra.
