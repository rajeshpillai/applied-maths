09 · Convex Optimisation

When the loss landscape has a single bottom — and how we know

Pre-flight check

  1. Have you done T5 L05 — Partial Derivatives and T5 L04 — Eigenvalues?
  2. Comfortable with gradients and the Hessian matrix (T4 L06 S1)?
  3. Recall positive-definite matrices (eigenvalues > 0)?

Touch

Pick a function. Draw a straight line between any two points on its graph. The widget tells you whether the curve always sits below that line — bowl-shaped — or whether it pokes through somewhere.

Wonder

  1. Why is “convex” so important for optimisation?
  2. Quadratic functions f(x) = (1/2) x^T A x + b^T x + c — when are they convex?
  3. Are sums of convex functions convex?

Discover

Problem 1

A function f: \mathbb{R} \to \mathbb{R} is convex iff f''(x) \ge 0 everywhere (for twice-differentiable f).

Verify f(x) = x^4 is convex.

Full solution

f''(x) = 12 x^2 \ge 0 ✓. Convex.

(Strictly convex except at x = 0.)

Problem 2

For multivariate f, convex iff Hessian \nabla^2 f is positive semi-definite (eigenvalues \ge 0).

Determine: is f(x, y) = x^2 + 2 x y + 3 y^2 convex?

Hint 1 — a nudge H = \begin{pmatrix} 2 & 2 \\ 2 & 6 \end{pmatrix}. Eigenvalues?
Full solution \det(H - \lambda I) = (2 - \lambda)(6 - \lambda) - 4 = \lambda^2 - 8 \lambda + 8 = 0 \Rightarrow \lambda = 4 \pm 2 \sqrt{2} \approx 6.83, 1.17. Both positive ⇒ Hessian PD ⇒ strictly convex.

Problem 3

Why convexity matters: every local minimum of a convex function is a global minimum.

Sketch: if x^* is a local min and y has f(y) < f(x^*), then on the segment from x^* to y, by convexity, f stays \le f(y) < f(x^*) near x^* — contradicting local min.

Why is this practically huge?

Full solution

No “saddle” or “local minimum” traps. Gradient descent (and many other algorithms) on convex functions reach the global optimum reliably. For non-convex problems (most real ML), you might get stuck.

This is why much of machine learning theory focuses on convex problems first: results there are clean, then extended (with caveats) to non-convex.

Problem 4

Operations preserving convexity:

  • Sum of convex functions is convex.
  • \max\{f_1, f_2\} of convex functions is convex.
  • Composition g \circ f: convex if g is convex non-decreasing and f is convex (e.g.).

Show: f(x) = \max(0, x) (ReLU) is convex.

Full solution

ReLU is the max of two convex functions: 0 (constant — convex) and x (linear — convex). Max of convex is convex.

ReLU is convex but not strictly convex (flat for x < 0). Still, it’s the standard activation in deep networks.

Problem 5

Convex optimisation problems have well-developed theory:

  • Linear Programming (LP): linear objective, linear constraints. Polynomial-time algorithms.
  • Quadratic Programming (QP): quadratic objective, linear constraints. Solved efficiently.
  • Convex programming (general): convex objective, convex constraints. Polynomial-time interior-point methods.

Why is “non-convex” a hard line in computer science?

Full solution

Non-convex problems may have many local minima, requiring exponential time in the worst case. Many NP-hard problems are non-convex.

But many practically important problems (LP, semidefinite, certain ML losses for shallow models) are convex. Modern ML is mostly non- convex — yet works well in practice via stochastic gradient methods and architectural tricks.

Understanding why non-convex deep learning works is one of the biggest open theoretical questions in AI.

Linear programming was born in wartime. Soviet mathematician Leonid Kantorovich invented LP in 1939 while consulting for the Leningrad plywood trust on how to allocate machines and raw timber optimally — convex optimisation, decades before the West had a name for it. His pamphlet circulated quietly through Soviet planning offices for years. In 1947 the American George Dantzig, working on US Air Force logistics, independently invented the simplex method for the same problem. Then in 1984, Narendra Karmarkar (at Bell Labs) shocked the field with the first polynomial-time interior-point algorithm, beating simplex’s exponential worst case and triggering the modern convex-optimisation renaissance. Kantorovich shared the 1975 Nobel Prize in Economics for his work.

A convex landscape is one where every “downhill” step is a step toward the global bottom. There are no false valleys, no traps, no surprises. This is the entire reason convex optimisation is clean: walking downhill always works, because there is only one “down.”

Name

A set C \subseteq \mathbb{R}^n is convex if for x, y \in C and t \in [0, 1]: (1 - t) x + t y \in C.

A function f is convex on C if for x, y \in C, t \in [0, 1]: f((1-t) x + t y) \le (1-t) f(x) + t f(y).

For twice-differentiable f:

  • 1D: convex iff f'' \ge 0.
  • \mathbb{R}^n: convex iff Hessian \nabla^2 f is positive semi-definite.

Strict convexity: strict inequality for t \in (0, 1), x \neq y.

Properties:

  • Local min = global min for convex f.
  • Sum, max, non-negative-scaling of convex are convex.
  • Composition with monotone functions sometimes preserves convexity.

Convex optimisation problem: \min_{x \in C} f(x), \quad C \text{ convex}, f \text{ convex}.

Common classes: linear (LP), quadratic (QP), semidefinite (SDP), general convex (interior-point methods).

Duality: primal problem ↔︎ dual problem; under regularity, equal optima.

Convex = lies below chords; convex set = closed under midpoints; Convex optimisation: local min = global min; tractable; Non-convex: hard, but ML works in practice via SGD.

Apply

Worked example

Show f(x) = e^x + x^2 is convex.

f''(x) = e^x + 2 > 0 for all x. Strictly convex.

(Sum of convex functions e^x and x^2 is convex.)

Practice — Warm

W1. Is f(x) = -x^2 convex?

Full solution f''(x) = -2 < 0. No (concave).

W2. Convex set: \{x : x \ge 0\}?

Full solution Yes (half-line).

W3. Convex set: union of two intervals?

Full solution Generally no (midpoint between intervals isn’t included).

Practice — Working

P1. Is the unit ball \{x : \|x\| \le 1\} convex?

Full solution Yes. By triangle inequality: \|t x + (1-t) y\| \le t \|x\| + (1-t) \|y\| \le 1.

P2. Convex Hessian for f(x, y) = 5 x^2 + 4 x y + 3 y^2?

Full solution H = \begin{pmatrix} 10 & 4 \\ 4 & 6 \end{pmatrix}. Det = 60 - 16 = 44 > 0; trace 16 > 0 ⇒ both eigenvalues positive ⇒ convex.

P3. Is the function f(x) = -\log x (for x > 0) convex?

Full solution f''(x) = 1/x^2 > 0. Convex. (Used in interior-point methods as a “barrier function” enforcing x > 0.)

Practice — Stretch

S1. Why is least squares regression convex? L(\beta) = \|y - X\beta\|^2.

Hint 1 — a nudge Quadratic in \beta: \beta^T X^T X \beta - 2 y^T X \beta + y^T y.
Full solution

L(\beta) = \beta^T (X^T X) \beta - 2 (X^T y)^T \beta + y^T y.

Hessian: 2 X^T X, which is positive semi-definite (Gram matrix). So L is convex.

Hence linear regression has a unique (or affine subspace of) minima — solvable in closed form (normal equations T5 L02 S2). This is why classical regression is so well-behaved.

S2. In modern ML, neural network losses are NOT convex. Why does training still work?

Full solution

Several theoretical reasons (active research):

  1. Over-parameterisation: enough capacity → loss landscape becomes “benign” (saddle points easy to escape, minima are wide and equivalent).
  2. Stochastic gradient noise: SGD’s noise helps escape saddles.
  3. Implicit bias: gradient descent finds particular nice minima (e.g., low-norm in linear models).
  4. Architecture: skip connections (ResNets) and normalisation (BatchNorm) shape the loss landscape favourably.
Despite non-convexity, modern deep nets reach (essentially) global minima reliably. Why is one of the biggest open theoretical questions in modern ML.

Common confusions

  1. “Convex set and convex function are the same.” Different things. A function is convex; its epigraph (region above its graph) is a convex set.
  2. “Convex = bowl-shaped only.” Higher dimensions: convex shapes include simplices, polyhedra, balls, ellipsoids — many forms.
  3. “All ML is non-convex.” Linear regression, logistic regression, SVM, many generative models are convex. Deep nets are not.

Takeaways

If you remember nothing else from this lesson, remember these:

  1. A convex function lies below every chord — bowl-shaped, no internal bumps; every local minimum is automatically the global minimum.
  2. The Hessian PSD test (eigenvalues \ge 0, via the eigenvalues of \nabla^2 f) is the workhorse check for convexity in higher dimensions.
  3. Convex problems are tractable; non-convex problems are not — yet modern deep learning works empirically because over- parameterisation, SGD noise, and architecture flatten the landscape.
