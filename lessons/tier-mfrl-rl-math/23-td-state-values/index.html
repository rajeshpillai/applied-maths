23 · TD(0): Bootstrapped Value Learning

Update v(s) using a bootstrapped target — immediate reward plus current estimate at next state.

Pre-flight check

Before you start, you should be able to say yes to all four:

  1. Can you state the Bellman equation for v_\pi(s)? — see MFRL — 06, Bellman Equation.
  2. Do you know what a Monte Carlo estimate of v_\pi(s) is — averaging full returns over episodes? — see MFRL — 17, MC Basic.
  3. Can you read the recursion w_{k+1} = (1 - \alpha) w_k + \alpha T_k as “old estimate plus step toward target”? — see MFRL — 21, Dvoretzky.
  4. Comfortable with discount factor \gamma \in [0,1) and discounted return G_t = r_{t+1} + \gamma r_{t+2} + \ldots?

If any answer is no, follow the link before continuing.

Touch

Below is a 5-state chain MDP. State 5 is the goal; reaching it gives reward 1. Other transitions give reward 0. The agent moves right with probability 0.8 and left with probability 0.2. Discount \gamma = 0.9.

The widget runs TD(0) value learning. Each step a transition arrives, and the agent updates one state’s value — by adding \alpha times the “TD error” (target minus current estimate). Watch the bars approach the true values (dashed grey).

viewof tdAlpha = Inputs.range([0.01, 0.5], {value: 0.1, step: 0.01, label: "α ="})
viewof numSteps = Inputs.range([10, 2000], {value: 400, step: 10, label: "Steps:"})
viewof tdSeed = Inputs.range([1, 50], {value: 4, step: 1, label: "Seed:"})
{
  function mulberry32(a) {
    return function() {
      a |= 0; a = a + 0x6D2B79F5 | 0;
      let t = a;
      t = Math.imul(t ^ t >>> 15, t | 1);
      t ^= t + Math.imul(t ^ t >>> 7, t | 61);
      return ((t ^ t >>> 14) >>> 0) / 4294967296;
    };
  }
  const rand = mulberry32(tdSeed * 11);
  const N = 5;          // states 0..4 (we'll display 1..5)
  const gamma = 0.9;
  const v = new Array(N).fill(0);
  let s = 0;
  for (let k = 0; k < numSteps; k++) {
    // Right with prob 0.8, left with prob 0.2.
    const right = rand() < 0.8;
    let s2 = s + (right ? 1 : -1);
    if (s2 < 0) s2 = 0;
    const reward = (s2 === N - 1) ? 1 : 0;
    // Treat goal as terminal: bootstrap v[N-1] = 0 once goal reached.
    const target = reward + gamma * (s2 === N - 1 ? 0 : v[s2]);
    v[s] = v[s] + tdAlpha * (target - v[s]);
    s = (s2 === N - 1) ? 0 : s2;
  }
  // True values via solving Bellman directly (rough closed form).
  // For this random-walk-style chain we use the analytic vπ found offline.
  const trueV = [0.405, 0.539, 0.685, 0.829, 0.000]; // computed numerically
  const data = v.map((x, i) => ({state: `s${i+1}`, est: x, truth: trueV[i]}));
  const Plot = await import("https://cdn.jsdelivr.net/npm/@observablehq/plot@0.6/+esm");
  return Plot.plot({
    width: 540, height: 280,
    x: {label: "state"},
    y: {label: "value", domain: [0, 1.0]},
    marks: [
      Plot.barY(data, {x: "state", y: "est", fill: "steelblue"}),
      Plot.tickY(data, {x: "state", y: "truth", stroke: "#999", strokeWidth: 2})
    ]
  });
}

Increase steps. Blue bars (TD estimates) converge to grey ticks (truth). The goal state is forced to value 0 as terminal. And the agent never computes a single full return.

Wonder

  1. The TD update uses v(s') — a current estimate, not the truth — to refine v(s). We are using a guess to fix another guess. Why doesn’t that loop diverge?
  2. Monte Carlo estimation needs full episodes; TD updates after every step. What does that buy you in continuing tasks (no terminal state)?
  3. The TD error \delta = r + \gamma v(s') - v(s) has a sign. What does the sign tell you about the current estimate?
  4. If v is already exact (v = v_\pi), what is the expected TD error?

Discover

Six problems. By the end you will see TD(0) as both Bellman and Robbins-Monro.

Problem 1 — the Monte Carlo target

Monte Carlo estimation updates v(s_t) \leftarrow v(s_t) + \alpha (G_t - v(s_t)), where G_t = r_{t+1} + \gamma r_{t+2} + \ldots is the actual return. What is the bias of G_t as an estimator of v_\pi(s_t)? What is its variance?

Hint 1 — a nudge \mathbb{E}_\pi[G_t \mid s_t] = v_\pi(s_t) by definition. Variance comes from many random rewards.
Full solution G_t is unbiased: \mathbb{E}[G_t \mid s_t] = v_\pi(s_t). Its variance is high: G_t is the sum of many random rewards, so its variance grows roughly like \sigma_r^2 / (1 - \gamma^2) for episode length H = 1/(1-\gamma).

Problem 2 — the TD target

Substitute the Bellman recursion G_t = r_{t+1} + \gamma G_{t+1} and replace the unknown G_{t+1} by the current estimate v(s_{t+1}). What target do you get?

Full solution The TD target: r_{t+1} + \gamma v(s_{t+1}). This is a one-step estimate using one real reward and one bootstrapped estimate.

Problem 3 — the TD update

Plug the TD target into the same shape: v(s) \leftarrow v(s) + \alpha (\text{target} - v(s)). Write the result.

Full solution v(s_t) \leftarrow v(s_t) + \alpha [r_{t+1} + \gamma v(s_{t+1}) - v(s_t)]. The bracket is called the TD error \delta_t = r_{t+1} + \gamma v(s_{t+1}) - v(s_t). The update is TD(0).

Problem 4 — bias and variance of the TD target

Compare TD target r_{t+1} + \gamma v(s_{t+1}) to MC target G_t. Which has higher variance? Which is biased?

Hint 1 — a nudge TD uses one real reward; MC uses many. TD uses the current estimate v (which is wrong while learning); MC uses none.
Full solution Variance: TD target uses one random reward, so its variance is \sigma_r^2 + \gamma^2 \mathrm{Var}(v(s_{t+1})) — much smaller than MC’s \sim \sigma_r^2 / (1-\gamma^2). Bias: TD target is biased while v \neq v_\pi, since it plugs in a wrong value at s_{t+1}. MC is unbiased. The trade is exact and fundamental: TD trades bias for variance.

Problem 5 — TD as Robbins-Monro

The Bellman equation says v_\pi(s) = \mathbb{E}_\pi[r + \gamma v_\pi(s') \mid s]. Write this as g(v_\pi(s)) = 0 for some g. Then identify the TD update as RM applied to this g.

Hint 1 — a nudge Move v_\pi(s) to one side: v_\pi(s) - \mathbb{E}_\pi[r + \gamma v_\pi(s')] = 0.
Full solution Let g(v(s)) = v(s) - \mathbb{E}_\pi[r + \gamma v(s')]. The Bellman fixed point satisfies g(v_\pi(s)) = 0. A noisy sample of g at v(s) is y = v(s) - (r + \gamma v(s')) where (r, s') is a sampled transition. The RM update is v(s) \leftarrow v(s) - \alpha y = v(s) + \alpha(r + \gamma v(s') - v(s)), which is TD(0). So TD = RM applied to the Bellman equation.

Problem 6 — convergence guarantee

State a sufficient condition (in words) for tabular TD(0) to converge to v_\pi.

Full solution Two conditions: (i) Every state visited infinitely often under the data-generating policy \pi (so each v(s) gets infinitely many updates). (ii) Step sizes \alpha_k for each state satisfy \sum \alpha_k = \infty and \sum \alpha_k^2 < \infty. Then v(s) \to v_\pi(s) a.s. — by Dvoretzky’s theorem, applied state-by-state with the Bellman operator’s contraction supplying the “contracting target” condition.

TD(0) is the marriage of two ideas you already own: the Bellman equation (which says v_\pi is a fixed point of the expected-Bellman operator), and Robbins-Monro (which solves fixed-point equations from samples). The “bootstrapping” word makes the marriage sound mysterious; the algebra is straightforward.

The name temporal-difference was coined by Richard Sutton in his 1988 paper Learning to Predict by the Methods of Temporal Differences. The same paper introduced TD(λ) — the eligibility-trace family that interpolates between TD(0) and Monte Carlo. TD-Gammon (Tesauro, 1992) used TD(λ) with a neural network and beat the world’s best human backgammon players. Modern policy-evaluation algorithms still descend from this 1988 paper.

Name

TD(0) — temporal-difference learning of the state-value function. Given a sample transition (s_t, r_{t+1}, s_{t+1}) collected under policy \pi, update v(s_t) \;\leftarrow\; v(s_t) + \alpha_t \, [\,\underbrace{r_{t+1} + \gamma v(s_{t+1})}_{\text{TD target}} - v(s_t)\,], where the bracket is the TD error \delta_t.

Under (i) every state visited infinitely often and (ii) \sum_t \alpha_t(s) = \infty, \sum_t \alpha_t(s)^2 < \infty for every state, v \to v_\pi almost surely (Dvoretzky + Bellman contraction).

A short verse, in the spirit of Bhāskara’s Līlāvatī:

The reward I just saw, plus my guess at next — That is my target, the bootstrap. I move toward it by alpha, And the error fades from minus to zero.

Apply

Worked example — TD(0) on a 3-state chain by hand

Three states s_1, s_2, s_3. Transitions: s_1 \to s_2 \to s_3 with reward 0 each; from s_3 the agent terminates with reward 1. Discount \gamma = 0.9. Initialise v(s_1) = v(s_2) = v(s_3) = 0. Step size \alpha = 0.5. Run one full episode and update the values along the way.

A competent solver thinks like this:

  1. Trajectory: s_1 \to s_2 (reward 0), s_2 \to s_3 (reward 0), s_3 \to \text{terminal} (reward 1).
  2. Update at s_1: target = 0 + 0.9 \cdot v(s_2) = 0. TD error = 0 - 0 = 0. New v(s_1) = 0 + 0.5 \cdot 0 = 0.
  3. Update at s_2: target = 0 + 0.9 \cdot v(s_3) = 0. TD error = 0 - 0 = 0. New v(s_2) = 0.
  4. Update at s_3: target = 1 + 0.9 \cdot 0 = 1 (terminal next). TD error = 1 - 0 = 1. New v(s_3) = 0 + 0.5 \cdot 1 = 0.5.

After one episode, only v(s_3) has moved! The reward “propagates backward” one state per episode. Over many episodes: - Episode 2: v(s_2) now sees a non-zero target 0 + 0.9 \cdot 0.5 = 0.45, so v(s_2) becomes 0 + 0.5 \cdot 0.45 = 0.225. And v(s_3) moves further toward 1. - This slow back-propagation of reward is one of TD’s quirks; it is exactly why n-step methods (Lesson 25) are often faster.

True values: v_\pi(s_3) = 1, v_\pi(s_2) = 0.9, v_\pi(s_1) = 0.81.

Practice — Warm

W1. What is the TD error if v already equals v_\pi?

Full solution \mathbb{E}[\delta_t] = \mathbb{E}[r + \gamma v_\pi(s')] - v_\pi(s) = 0 by the Bellman equation. In expectation the TD error is zero at convergence. A single sampled \delta_t may not be zero — that’s the noise that drives the iteration.

W2. Compute the TD update for one transition with r = 2, v(s) = 1.5, v(s') = 1.0, \gamma = 0.9, \alpha = 0.1.

Full solution Target = 2 + 0.9 \cdot 1.0 = 2.9. TD error = 2.9 - 1.5 = 1.4. v(s)_{\text{new}} = 1.5 + 0.1 \cdot 1.4 = 1.64.

W3. Constant \alpha = 0.1 versus \alpha_t = 1/t (per state). Which gives a.s. convergence? Which gives tracking?

Full solution \alpha_t = 1/t gives a.s. convergence (RM conditions). \alpha = 0.1 gives tracking — the iterate orbits v_\pi in a noise cloud forever. Practical implementations use constant \alpha because the policy is also changing (non-stationary target).

Practice — Working

P1. A 4-state cycle: s_1 \to s_2 \to s_3 \to s_4 \to s_1, all rewards 0 except s_4 \to s_1 which gives +1. \gamma = 0.9, \alpha = 0.2, all v = 0 initially. Compute v after one full lap (4 transitions).

Hint 1 — a nudge Update each state in turn using the TD error.
Full solution

Transitions and updates (using current v at the time of each update): - s_1 \to s_2, r=0: \delta = 0 + 0.9 \cdot 0 - 0 = 0, v(s_1) = 0. - s_2 \to s_3, r=0: \delta = 0, v(s_2) = 0. - s_3 \to s_4, r=0: \delta = 0, v(s_3) = 0. - s_4 \to s_1, r=1: \delta = 1 + 0.9 \cdot 0 - 0 = 1, v(s_4) = 0 + 0.2 \cdot 1 = 0.2.

Only v(s_4) moves on lap 1. Each successive lap pushes the reward back one state. The true infinite-horizon values are v_\pi(s_4) \approx 0.778, v_\pi(s_3) \approx 0.700, v_\pi(s_2) \approx 0.630, v_\pi(s_1) \approx 0.567.

P2. Compare TD(0) and MC on a noisy reward channel. Suppose every transition reward is 0 on average but has variance \sigma^2 = 1. Episode length is H = 100. Which method gives a lower-variance per-state estimate per episode?

Hint 1 — a nudge MC: variance of the full H-reward sum. TD(0): variance of one reward.
Full solution MC’s full-return variance is H \sigma^2 = 100. TD(0)’s per-step target variance is \approx \sigma^2 + \gamma^2 \mathrm{Var}(v(s')) \approx 1. TD has 100× lower variance per update. This is why TD wins on long or continuing tasks — MC’s variance scales with horizon length.

P3. Show that TD(0) with constant \alpha converges in expectation to v_\pi (not necessarily a.s.). Sketch.

Hint 1 — a nudge Take expectations of the TD update; use the Bellman fixed point.
Full solution \mathbb{E}[v_{k+1}(s)] = (1 - \alpha) \mathbb{E}[v_k(s)] + \alpha \mathbb{E}[r + \gamma v_k(s')]. The expected-Bellman operator is a \gamma-contraction in \ell_\infty, so the expectation contracts to v_\pi at rate 1 - \alpha(1 - \gamma). In expectation, v converges geometrically. The almost-sure statement requires vanishing \alpha_k because the per-step variance \alpha^2 \sigma^2 never dies otherwise.

Practice — Stretch

S1. Why doesn’t TD diverge when it uses a guess to update a guess? Walk through the contraction-plus-Dvoretzky argument.

Hint 1 — a nudge Two ingredients: contraction of the expected operator, and bounded variance of the noisy sample.
Full solution Let \mathcal{T}_\pi be the expected-Bellman operator (\mathcal{T}_\pi v)(s) = \mathbb{E}_\pi[r + \gamma v(s') \mid s]. It is a \gamma-contraction in \ell_\infty, with unique fixed point v_\pi. The TD target is a noisy unbiased estimate of \mathcal{T}_\pi v at s. Dvoretzky’s theorem (Lesson 21) handles exactly this: a contracting target plus RM step conditions yields convergence. The “guess updating a guess” worry vanishes because the expected update is a contraction; the noise is just noise.

S2. Suppose two states s_1, s_2 are visited equally often; s_3 is visited 10× less. With constant \alpha, who learns faster — and what does that imply for exploration?

Full solution s_1, s_2 each get effective effective step size \alpha per visit; s_3 gets the same per-visit step but only 1/10 the visits. So s_3’s estimate moves 10\times slower in wall-clock terms. For accurate value estimates under a fixed \alpha, you need balanced state-visitation. This is why exploration policies (ε-greedy, soft policies) matter for accurate value-learning, not just for finding optimal actions.

Common confusions

  1. “TD updates a guess with a guess, so it must be wrong.” No — it’s biased during learning, but the bias shrinks as v improves. In the limit v = v_\pi the target is unbiased, and the iteration has reached its fixed point.
  2. “TD is just MC with a smaller window.” Subtly different. MC averages full random returns; TD substitutes a current estimate into a one-step target. The first uses no model assumption; the second assumes the Bellman recursion holds.
  3. “You need to wait for episodes to end.” Not for TD. That’s MC’s problem. TD updates after every transition — usable in continuing tasks with no terminal state.

Takeaways

If you remember nothing else:

  1. TD update: v(s) \leftarrow v(s) + \alpha [r + \gamma v(s') - v(s)]. The bracket is the TD error.
  2. TD = Bellman fixed point + Robbins-Monro. The bootstrapped target is a noisy sample of the expected-Bellman operator.
  3. TD trades variance for bias. Lower per-step variance than MC; biased while learning. For long horizons, TD wins.
