22 · SGD: Gradient Descent with Sampled Gradients

SGD as a special case of Robbins-Monro applied to g(w) = ∇f(w) = 0.

Pre-flight check

Before you start, you should be able to say yes to all four:

  1. Can you state full-batch gradient descent w \leftarrow w - \eta \nabla L(w) in one breath? — see Tier 6, Lesson 10 — Gradient Methods.
  2. Can you compute a gradient \nabla_w f(w) for a simple quadratic? — same prerequisite.
  3. Have you internalised Dvoretzky-shaped iterations? — see MFRL — 21, Dvoretzky.
  4. Can you state RM step conditions \sum \alpha_k = \infty, \sum \alpha_k^2 < \infty? — see MFRL — 20, Robbins-Monro.

If any answer is no, follow the link before continuing.

Touch

Imagine you are walking downhill in fog. You can compute the slope under your feet, but only by averaging many footsteps’ worth of soil-tilt — expensive. Or you can pick one random footstep and trust it as a noisy estimate of the local slope — cheap, but jittery.

Drag the slider to switch between full-batch gradient descent (averages all training points each step — smooth) and stochastic gradient descent (uses one point at a time — zigzag). The loss is the convex L(w) = \tfrac{1}{n}\sum_{i=1}^n \tfrac{1}{2}(w - x_i)^2 for a fixed dataset. The minimiser is w^\* = \bar x.

viewof method = Inputs.select(["Full-batch GD", "Mini-batch (b=4)", "SGD (b=1)"], {label: "Gradient method:"})
viewof eta = Inputs.range([0.01, 0.5], {value: 0.15, step: 0.01, label: "Step η:"})
viewof seedSGD = Inputs.range([1, 60], {value: 8, step: 1, label: "Seed:"})
{
  function mulberry32(a) {
    return function() {
      a |= 0; a = a + 0x6D2B79F5 | 0;
      let t = a;
      t = Math.imul(t ^ t >>> 15, t | 1);
      t ^= t + Math.imul(t ^ t >>> 7, t | 61);
      return ((t ^ t >>> 14) >>> 0) / 4294967296;
    };
  }
  const rand = mulberry32(seedSGD * 7);
  // Make a fixed dataset of 30 points roughly centered at 3, with spread 2.
  const n = 30;
  const xs = [];
  for (let i = 0; i < n; i++) {
    const u1 = Math.max(rand(), 1e-9), u2 = rand();
    const z = Math.sqrt(-2 * Math.log(u1)) * Math.cos(2 * Math.PI * u2);
    xs.push(3 + 2 * z);
  }
  const xbar = xs.reduce((a, b) => a + b, 0) / n;

  const b = method.startsWith("Full") ? n : (method.startsWith("Mini") ? 4 : 1);

  let w = -2;
  const series = [{k: 0, w}];
  for (let k = 1; k <= 80; k++) {
    // Sample b points; gradient is (w - xi); average across batch.
    let g = 0;
    for (let j = 0; j < b; j++) {
      const idx = Math.floor(rand() * n);
      g += (w - xs[idx]);
    }
    g /= b;
    w = w - eta * g;
    series.push({k, w});
  }
  const Plot = await import("https://cdn.jsdelivr.net/npm/@observablehq/plot@0.6/+esm");
  return Plot.plot({
    width: 540, height: 280,
    x: {label: "iteration k"},
    y: {label: "w_k", domain: [-3, 6]},
    marks: [
      Plot.ruleY([xbar], {stroke: "#999", strokeDasharray: "4 3"}),
      Plot.line(series, {x: "k", y: "w", stroke: "steelblue"}),
      Plot.text([{k: 70, w: xbar}], {x: "k", y: "w", text: ["w* = mean"], dy: -10, fontSize: 11, fill: "#666"})
    ]
  });
}

The grey dashed line is the true minimiser w^\* = \bar x. Full-batch is a smooth slide. SGD zig-zags. Mini-batch is in between. They all eventually arrive — the question is at what cost.

Wonder

  1. The full-batch gradient is the truth. Why would anyone deliberately replace it with a noisy estimate?
  2. SGD’s zigzag wastes some of every step. Can the zigzag ever be a feature — something that helps, not hurts?
  3. Mini-batch with b=4 uses four times the compute of b=1 for one step, but quarters the variance per step. Which is the better deal?
  4. We just spent two lessons proving that noise can still converge. Is SGD really new, or is it Robbins-Monro wearing different clothes?

Discover

Six problems that walk from full-batch GD to SGD and prove they are the same algorithm with different noise.

Problem 1 — full-batch GD on a finite dataset

Define L(w) = \tfrac{1}{n}\sum_{i=1}^n \ell_i(w) for losses \ell_1, \ldots, \ell_n. Write the gradient \nabla L(w) explicitly.

Full solution \nabla L(w) = \tfrac{1}{n}\sum_{i=1}^n \nabla \ell_i(w). The full-batch gradient is the average of the per-sample gradients.

Problem 2 — average vs random sample

Suppose I pick an index I_k uniformly at random from \{1,\ldots,n\}. What is \mathbb{E}[\nabla \ell_{I_k}(w)]?

Hint 1 — a nudge Uniform expectation = average.
Full solution \mathbb{E}[\nabla \ell_{I_k}(w)] = \tfrac{1}{n}\sum_{i=1}^n \nabla \ell_i(w) = \nabla L(w). One random per-sample gradient is an unbiased estimator of the full gradient.

Problem 3 — the SGD update

Replace \nabla L(w_k) in the GD update by its random estimator. Write the resulting iteration.

Full solution w_{k+1} = w_k - \alpha_k \nabla \ell_{I_k}(w_k). This is stochastic gradient descent (SGD). Per step it touches one data point, not n.

Problem 4 — SGD as a Robbins-Monro instance

Let g(w) = \nabla L(w). The minimum of L is a root of g. Show that the SGD update is RM applied to this g.

Hint 1 — a nudge Recall RM: w_{k+1} = w_k - \alpha_k y_k with y_k a noisy sample of g(w_k).
Full solution The noisy sample is y_k = \nabla \ell_{I_k}(w_k) = \nabla L(w_k) + \eta_k where \eta_k = \nabla \ell_{I_k}(w_k) - \nabla L(w_k) is mean-zero by Problem 2. So w_{k+1} = w_k - \alpha_k y_k is RM. SGD = RM(∇L).

Problem 5 — mini-batch as variance control

If you use a mini-batch of size b instead of one sample, what happens to the noise variance per step?

Hint 1 — a nudge Variance of an average of b i.i.d. things scales as…
Full solution The noise variance falls as \sigma^2 / b. So mini-batch b=4 reduces per-step noise by 4× — but costs 4× compute per step. Net rate is roughly unchanged, but mini-batch (i) parallelises beautifully on GPUs, (ii) gives less zigzag, and (iii) keeps an unbiased gradient.

Problem 6 — convergence by Dvoretzky

State the Dvoretzky-style argument: for what schedules \alpha_k does SGD converge to a stationary point of L?

Hint 1 — a nudge Step conditions \sum \alpha_k = \infty, \sum \alpha_k^2 < \infty, plus some smoothness on L.
Full solution For convex L with bounded gradient variance, the schedule \alpha_k = c/k (or any RM-compatible sequence) gives w_k \to w^\* a.s. For non-convex L, the same schedule gives convergence of \nabla L(w_k) \to 0 a.s. — i.e., to a stationary point, not necessarily a global minimum. The proof is “verify Dvoretzky-shape, quote the theorem.”

SGD is not a gradient-descent variant; it is Robbins-Monro applied to the gradient. Every property of SGD — convergence, noise behaviour, rate — is inherited from stochastic approximation theory.

The 1986 backprop paper by Rumelhart, Hinton, and Williams trained neural networks with mini-batch SGD. Before that, neural-network training used full-batch GD and didn’t scale past toy problems. The shift to SGD, twinned with the rediscovery of backprop, is when neural networks became possible. The 2012 AlexNet paper that ignited deep learning trained on ImageNet with mini-batch SGD on two GPUs.

Name

Stochastic gradient descent (SGD). Given L(w) = \tfrac{1}{n}\sum_i \ell_i(w), sample I_k \sim \mathrm{Uniform}\{1,\ldots,n\} and update w_{k+1} = w_k - \alpha_k \, \nabla \ell_{I_k}(w_k).

  • Full-batch GD: uses \nabla L(w_k) = \tfrac{1}{n}\sum_i \nabla \ell_i(w_k) exactly.
  • Mini-batch SGD: averages over a batch of size b: \hat\nabla = \tfrac{1}{b}\sum_{j=1}^b \nabla \ell_{I_{k,j}}(w_k).
  • SGD: is mini-batch with b = 1.

All three are unbiased estimators of \nabla L. Convergence to a stationary point of L holds under Robbins-Monro step conditions plus mild smoothness — a corollary of Dvoretzky.

A short verse, in the spirit of Bhāskara’s Līlāvatī:

One sample’s gradient — noisy but free. Many samples averaged — true but slow. Step shrinks like one over k — both arrive. Noise is the price; speed is the reward.

Apply

Worked example — SGD on linear regression by hand

Fit \hat y = w x to the dataset (x_i, y_i) = (1, 2), (2, 5), (3, 7). The squared-error loss is L(w) = \tfrac{1}{6}\sum_{i=1}^3 (w x_i - y_i)^2 with gradient \nabla L(w) = \tfrac{1}{3}\sum_{i=1}^3 (w x_i - y_i) x_i. Run SGD from w_0 = 0 with \alpha_k = 1/(k+1) on the sample sequence I_1 = 2, I_2 = 1, I_3 = 3 (deterministic for this exercise).

A competent solver thinks like this:

  1. Per-sample gradient: \nabla \ell_i(w) = (w x_i - y_i) x_i.
  2. Step 1: I_1 = 2, sample is (2, 5). Gradient at w_0 = 0 is (0 \cdot 2 - 5) \cdot 2 = -10. Update: w_1 = 0 - \tfrac{1}{2} \cdot (-10) = 5.
  3. Step 2: I_2 = 1, sample is (1, 2). Gradient at w_1 = 5 is (5 \cdot 1 - 2) \cdot 1 = 3. Update: w_2 = 5 - \tfrac{1}{3} \cdot 3 = 4.
  4. Step 3: I_3 = 3, sample is (3, 7). Gradient at w_2 = 4 is (4 \cdot 3 - 7) \cdot 3 = 15. Update: w_3 = 4 - \tfrac{1}{4} \cdot 15 = 0.25.

Estimate after 3 SGD steps: \hat w = 0.25. Sanity check: the closed-form least-squares solution is w^\* = \sum x_i y_i / \sum x_i^2 = (2+10+21)/(1+4+9) = 33/14 \approx 2.36. Three SGD steps from a bad initialisation aren’t enough to converge — but the iteration is doing exactly the right work, and a hundred steps would close the gap.

Practice — Warm

W1. A friend writes “SGD with \alpha = 0.01 converges by Dvoretzky.” Spot the bug.

Full solution A constant \alpha violates \sum \alpha_k^2 < \infty. SGD with constant \alpha tracks the optimum with persistent noise of size \sim \alpha \sigma. It does not converge a.s. (See Lesson 21.)

W2. For squared-error loss \ell_i(w) = \tfrac12 (w - x_i)^2, write the SGD update.

Full solution \nabla \ell_i(w) = w - x_i. Update: w_{k+1} = w_k - \alpha_k (w_k - x_{I_k}) = (1 - \alpha_k) w_k + \alpha_k x_{I_k}. This is exactly the running-mean estimator from Lesson 16. Mean estimation is SGD on squared error.

W3. For batch size b, what is the noise variance reduction relative to single-sample SGD?

Full solution Variance falls by a factor of b. Standard error falls by \sqrt{b}. So b=4 halves the standard error per step.

Practice — Working

P1. Compute the SGD update for logistic regression. Loss \ell_i(w) = \log(1 + e^{-y_i w^\top x_i}) with y_i \in \{-1, +1\}. What is the per-sample gradient?

Hint 1 — a nudge \frac{d}{dw}\log(1+e^{-yw^\top x}) = \frac{-y x e^{-yw^\top x}}{1 + e^{-yw^\top x}}.
Full solution \nabla \ell_i(w) = -y_i x_i \, \sigma(-y_i w^\top x_i) where \sigma(z) = 1/(1+e^{-z}) is the logistic function. SGD update: w_{k+1} = w_k + \alpha_k y_{I_k} x_{I_k} \sigma(-y_{I_k} w_k^\top x_{I_k}). This is the engine of every logistic-regression library.

P2. Show that mini-batch SGD with batch size b is also a Dvoretzky iteration. Identify \alpha_k, \beta_k, T_k, w^\*.

Hint 1 — a nudge The recursion is w_{k+1} = w_k - \alpha_k \hat\nabla_k with \hat\nabla_k = \frac{1}{b}\sum_{j=1}^b \nabla \ell_{I_{k,j}}(w_k).
Full solution Reshape: w_{k+1} = w_k - \alpha_k \hat\nabla_k. To put it in the master form, treat T_k = w_k - \hat\nabla_k, \alpha_k = \beta_k = \alpha_k: w_{k+1} = (1-\alpha_k) w_k + \alpha_k T_k. The fixed point of the iteration is w^\* where \nabla L(w^\*) = 0. Conditional on \mathcal{F}_k, \mathbb{E}[\hat\nabla_k] = \nabla L(w_k), so \mathbb{E}[T_k] = w_k - \nabla L(w_k) — a contracting target as long as we have local strong convexity. Dvoretzky applies.

P3. The variance question. SGD’s per-step variance is \sigma^2 / b. Compare two strategies for fixed total compute T samples: (a) one full-batch step with b = T; (b) T SGD steps with b = 1. Which makes more progress on a strongly convex objective?

Hint 1 — a nudge Full-batch contracts per step by a constant; SGD contracts on average per step by a constant.
Full solution On strongly convex L with condition number \kappa: full-batch T = b gives one contraction by 1 - 1/\kappa. SGD takes T steps, each contracting on average by 1 - O(1/\kappa), accumulating roughly (1 - 1/\kappa)^T \approx \exp(-T/\kappa) — much better. SGD wins when the dataset is large because it can take many noisy steps in the time it takes full-batch to take one clean one.

Practice — Stretch

S1. Why does SGD escape saddle points that GD gets stuck in? Sketch.

Hint 1 — a nudge At a saddle point, the gradient is zero. GD stops. What does SGD do?
Full solution At a saddle, \nabla L = 0 so GD makes no progress. SGD’s per-sample gradient is non-zero — its expectation is zero, but its realisation isn’t. The noise gives SGD a random push, and any push along a negative-curvature direction sets the iterate sliding off the saddle. The 2015 paper Escaping from Saddle Points — Online Stochastic Gradient for Tensor Decomposition (Ge et al.) made this precise.

S2. Reconcile: Dvoretzky says SGD with constant \alpha does not converge, yet every modern deep-learning paper uses constant or piecewise-constant learning rates. How can both be true?

Hint 1 — a nudge What does “converge” mean here? Two senses are at play.
Full solution “Convergence almost surely” needs vanishing \alpha. Constant-\alpha SGD does converge — but in the weaker sense of a stationary distribution: the iterate orbits the optimum in a noise cloud of size O(\sqrt{\alpha} \cdot \sigma). For deep networks, this orbiting region happens to be of similar size to the basin of generalisation; you do not actually want to “land” at w^\* exactly because w^\* on the training set is not w^\* on the test set. So practitioners stay in the noise cloud deliberately. Theory and practice are consistent once you read “convergence” carefully.

Common confusions

  1. “SGD is biased.” The expected gradient over a uniformly sampled index is the full gradient. SGD is unbiased; it just has variance.
  2. “Constant \alpha doesn’t work.” It works in practice for tracking and for non-stationary objectives; it just does not satisfy the formal a.s.-convergence theorem. Pick a step rule that matches your goal.
  3. “SGD = GD with bigger steps.” No. SGD is GD with a random gradient. The noise is structurally different from a step-size change; it’s the noise that lets SGD escape saddle points.

Takeaways

If you remember nothing else:

  1. SGD is RM applied to g(w) = \nabla L(w) = 0. Every property follows.
  2. Mini-batch is a variance knob: higher b = lower noise per step = higher cost per step.
  3. Constant \alpha tracks; vanishing \alpha converges. Pick based on whether you have a stationary or moving target.
