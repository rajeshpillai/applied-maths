25 · n-Step Sarsa: Bridging TD(0) and Monte Carlo

Bootstrap after n steps instead of 1: the bias-variance knob between TD and MC.

Pre-flight check

Before you start, you should be able to say yes to all four:

  1. Can you write the Sarsa update from memory? — see MFRL — 24, Sarsa.
  2. Can you state the Monte Carlo target — full discounted return G_t? — see MFRL — 17, MC Basic.
  3. Comfortable with the bias/variance trade in TD vs MC? — recap in MFRL — 23, TD(0) Discover Problem 4.
  4. Discount-factor identity G_t = r_{t+1} + \gamma r_{t+2} + \gamma^2 r_{t+3} + \ldots?

If any answer is no, follow the link before continuing.

Touch

Pick n — the number of steps before the bootstrap target kicks in. n = 1 is plain Sarsa; n = \infty is Monte Carlo. Anything in between is n-step Sarsa.

The widget runs n-step Sarsa on a 10-state chain (start at left, terminal +1 reward at right, \gamma = 0.9, \alpha = 0.2). The blue line is the running RMS error of q versus q^\* over 200 episodes. Sweep n and watch how the error curve changes.

viewof nStep = Inputs.range([1, 30], {value: 4, step: 1, label: "n ="})
viewof nSeed = Inputs.range([1, 30], {value: 9, step: 1, label: "Seed:"})
{
  function mulberry32(a) {
    return function() {
      a |= 0; a = a + 0x6D2B79F5 | 0;
      let t = a;
      t = Math.imul(t ^ t >>> 15, t | 1);
      t ^= t + Math.imul(t ^ t >>> 7, t | 61);
      return ((t ^ t >>> 14) >>> 0) / 4294967296;
    };
  }
  const rand = mulberry32(nSeed * 19);
  const N = 10;            // states 0..9; goal at 9
  const A = 2;             // 0=left,1=right
  const gamma = 0.9, alpha = 0.2;
  const q = new Array(N * A).fill(0);
  const idx = (s, a) => s * A + a;
  function chooseAction(s) {
    if (rand() < 0.1) return Math.floor(rand() * A);
    return q[idx(s, 0)] >= q[idx(s, 1)] ? 0 : 1;
  }
  function step(s, a) {
    const ns = a === 0 ? Math.max(0, s - 1) : Math.min(N - 1, s + 1);
    const reward = ns === N - 1 ? 1 : 0;
    const done = ns === N - 1;
    return {ns, reward, done};
  }
  // Approximate q* by running for many episodes with n=1 first as truth.
  const qStar = new Array(N * A).fill(0);
  for (let s = 0; s < N - 1; s++) for (let a = 0; a < A; a++) {
    // Greedy expected return: from state s, going right repeatedly.
    const dist = (N - 1) - s;
    qStar[idx(s, a)] = a === 1 ? Math.pow(gamma, dist - 1) : Math.pow(gamma, dist + 1);
  }
  const errors = [];
  for (let ep = 0; ep < 200; ep++) {
    let s = 0, a = chooseAction(s);
    const traj = [{s, a, r: 0}];
    let T = Infinity;
    let t = 0;
    while (true) {
      if (t < T) {
        const {ns, reward, done} = step(traj[t].s, traj[t].a);
        const na = done ? 0 : chooseAction(ns);
        traj.push({s: ns, a: na, r: reward});
        if (done) T = t + 1;
      }
      const tau = t - nStep + 1;
      if (tau >= 0) {
        let G = 0;
        for (let i = tau + 1; i <= Math.min(tau + nStep, T); i++) {
          G += Math.pow(gamma, i - tau - 1) * traj[i].r;
        }
        if (tau + nStep < T) {
          G += Math.pow(gamma, nStep) * q[idx(traj[tau + nStep].s, traj[tau + nStep].a)];
        }
        const i0 = idx(traj[tau].s, traj[tau].a);
        q[i0] = q[i0] + alpha * (G - q[i0]);
      }
      if (tau === T - 1) break;
      t++;
      if (t > 200) break;
    }
    let err2 = 0, count = 0;
    for (let s = 0; s < N - 1; s++) for (let a = 0; a < A; a++) {
      const e = q[idx(s, a)] - qStar[idx(s, a)];
      err2 += e * e; count++;
    }
    errors.push({ep, rms: Math.sqrt(err2 / count)});
  }
  const Plot = await import("https://cdn.jsdelivr.net/npm/@observablehq/plot@0.6/+esm");
  return Plot.plot({
    width: 540, height: 280,
    x: {label: "episode"},
    y: {label: "RMS error vs q*", domain: [0, 1.2]},
    marks: [
      Plot.line(errors, {x: "ep", y: "rms", stroke: "steelblue"})
    ]
  });
}

Try n = 1 (Sarsa), n = 4 (the Sutton-Barto sweet spot for this kind of chain), n = 10 (close to MC). n in the middle usually wins.

Wonder

  1. With n = 1, the reward at the goal must propagate back one state per episode. With n = 5, how far does the reward propagate per episode?
  2. Larger n means more real rewards in the target — and fewer guesses. Why might that increase variance?
  3. The optimal n depends on how noisy the environment is. Why?
  4. If you average over all n values weighted by \lambda^{n-1}, what famous algorithm do you get? (You don’t need to know — just guess from the shape.)

Discover

Six problems to derive the n-step return and connect TD to MC.

Problem 1 — the n-step return

Recall G_t = r_{t+1} + \gamma r_{t+2} + \gamma^2 r_{t+3} + \ldots. Truncate at n steps and bootstrap with q at step t + n. Write the result.

Full solution G_t^{(n)} = r_{t+1} + \gamma r_{t+2} + \cdots + \gamma^{n-1} r_{t+n} + \gamma^n q(s_{t+n}, a_{t+n}). This is the n-step return. Real rewards for n steps, then bootstrap.

Problem 2 — recover Sarsa and MC

What does G_t^{(n)} become when n = 1? When n \to \infty (assuming the episode terminates)?

Full solution n = 1: G_t^{(1)} = r_{t+1} + \gamma q(s_{t+1}, a_{t+1}) — the Sarsa target. n \to \infty: G_t^{(\infty)} = r_{t+1} + \gamma r_{t+2} + \ldots = G_t — the Monte Carlo target. n-step Sarsa interpolates exactly between these two.

Problem 3 — the n-step Sarsa update

Drop G_t^{(n)} in as the target.

Full solution q(s_t, a_t) \;\leftarrow\; q(s_t, a_t) + \alpha \, [G_t^{(n)} - q(s_t, a_t)]. Implementation note: you can only perform this update after you have observed n more transitions, so updates lag by n steps.

Problem 4 — the bias-variance knob

State qualitatively how bias and variance of G_t^{(n)} depend on n.

Hint 1 — a nudge Bias comes from bootstrapping q (which is wrong); variance comes from real random rewards.
Full solution Bias: decreases with n — more real rewards crowd out the biased bootstrap. At n = \infty (MC) bias = 0. Variance: increases with n — each extra reward injects its own randomness. At n = 1 variance ≈ \sigma_r^2; at n = \infty variance is O(\sigma_r^2 / (1-\gamma^2)). The optimal n balances the two.

Problem 5 — reward propagation

Imagine a chain of length L with reward only at the end. With n = 1 Sarsa, how many episodes are needed for the reward signal to reach state 0? With n-step Sarsa?

Full solution n = 1: each episode propagates the reward signal back one state, so L - 1 episodes are needed. n-step: each episode propagates back n states (the bootstrap target now sees the reward directly when within n of the goal). So \lceil (L - 1) / n \rceil episodes. n-step is faster on long chains.

Problem 6 — implementation memory

How many time-step tuples do you need to keep in a buffer to compute the n-step update at time t?

Full solution You need the last n + 1 tuples: the state-action (s_t, a_t) being updated, plus n subsequent rewards r_{t+1}, \ldots, r_{t+n}, plus the next state-action (s_{t+n}, a_{t+n}). Memory: O(n), much smaller than storing whole episodes for MC.

n is a knob that interpolates between TD’s low-variance bias and MC’s zero-bias variance. The middle of the knob almost always beats the endpoints — empirically on most benchmarks the sweet spot is n \in [3, 10] depending on horizon length.

Sutton’s 1988 TD(\lambda) paper introduced an even cleverer construction: exponentially weight all n-step returns by (1 - \lambda) \lambda^{n-1}. This is the λ-return. \lambda = 0 recovers TD(0); \lambda = 1 recovers MC; intermediate \lambda has the same sweet spot n-step has, but without picking a single n. The implementation uses eligibility traces — a clever bookkeeping trick that avoids storing all the n-step returns explicitly. We touch on this in the synthesis lesson but not in this track.

Name

n-step return: G_t^{(n)} \;=\; r_{t+1} + \gamma r_{t+2} + \cdots + \gamma^{n-1} r_{t+n} + \gamma^n \, q(s_{t+n}, a_{t+n}).

n-step Sarsa update: q(s_t, a_t) \;\leftarrow\; q(s_t, a_t) + \alpha \, [G_t^{(n)} - q(s_t, a_t)].

Special cases: n = 1 recovers Sarsa; n = T - t (full episode) recovers Monte Carlo.

A short verse, in the spirit of Bhāskara’s Līlāvatī:

Real rewards for n steps, then a bootstrap — Each step replaces a guess with a fact. Small n: low variance, high bias; Large n: low bias, high variance. Pick n to balance the two.

Apply

Worked example — 3-step Sarsa update on a chain

A chain s_1 \to s_2 \to s_3 \to s_4 \to s_5 (\text{terminal}) with rewards r_2 = 0, r_3 = 0, r_4 = 0, r_5 = +1. \gamma = 0.9, \alpha = 0.5. Initial q are all 0. Use n = 3 to update q(s_1, \text{right}) after observing the first 3 rewards.

A competent solver thinks like this:

  1. Buffer the trajectory: (s_1, \text{right}), then rewards r_2 = 0, r_3 = 0, r_4 = 0, arriving at (s_4, a_4) with a_4 = \text{right}.
  2. Compute G_1^{(3)}: 0 + 0.9 \cdot 0 + 0.9^2 \cdot 0 + 0.9^3 \cdot q(s_4, a_4) = 0. (Bootstrap is zero because q is initialised to zero.)
  3. Update q(s_1, \text{right}): 0 + 0.5 \cdot (0 - 0) = 0. No movement. The reward at the goal isn’t reached by the 3-step window from s_1.
  4. Continuing: by the time we update q(s_2, \text{right}) (after rewards r_3, r_4, r_5), the 3-step return is 0 + 0 + 0.81 \cdot 1 = 0.81, so q(s_2, \text{right}) \leftarrow 0 + 0.5 \cdot 0.81 = 0.405.

So 3-step Sarsa propagates the reward 3 states back per episode — versus 1 state for plain Sarsa. On this 5-state chain, 3-step finishes in 1 episode instead of 4.

Practice — Warm

W1. Compute G_t^{(2)} given r_{t+1} = 1, r_{t+2} = 2, q(s_{t+2}, a_{t+2}) = 5, \gamma = 0.5.

Full solution G_t^{(2)} = 1 + 0.5 \cdot 2 + 0.25 \cdot 5 = 1 + 1 + 1.25 = 3.25.

W2. Recover the Sarsa target by setting n = 1 in G_t^{(n)}.

Full solution G_t^{(1)} = r_{t+1} + \gamma q(s_{t+1}, a_{t+1}) — the Sarsa target.

W3. Show that as n \to \infty (with episode length T), G_t^{(n)} \to G_t.

Full solution For n \ge T - t, the bootstrap term \gamma^n q(s_{t+n}, a_{t+n}) becomes \gamma^{T-t} q(\text{terminal}) = 0 by convention (terminal = 0). The remaining sum is the full discounted return G_t. So n-step recovers MC at the largest n.

Practice — Working

P1. A chain with rewards r_2 = -1, r_3 = -1, r_4 = 10 (terminal). \gamma = 0.9. With n = 2, compute G_2^{(2)} assuming q(s_4, \cdot) = 0 (terminal).

Full solution G_2^{(2)} = r_3 + \gamma r_4 + \gamma^2 q(s_4, a_4) = -1 + 0.9 \cdot 10 + 0.81 \cdot 0 = 8.

P2. Bias-variance trade quantitatively. Suppose every reward has variance \sigma^2 = 1 and the current q has error |q(s, a) - q^\*(s, a)| \le \beta uniformly. Bound the variance and bias of G_t^{(n)} in terms of n, \gamma, \sigma, \beta.

Hint 1 — a nudge Variance: sum of n independent reward variances scaled by \gamma^{2k} plus bootstrap variance. Bias: error of q scaled by \gamma^n.
Full solution

\mathrm{Var}(G_t^{(n)}) = \sigma^2 \sum_{k=0}^{n-1} \gamma^{2k} + \gamma^{2n} \mathrm{Var}(q(s_{t+n}, a_{t+n})) \le \sigma^2 \frac{1 - \gamma^{2n}}{1 - \gamma^2} + \gamma^{2n} \beta^2. |\text{Bias}| = |\gamma^n (\mathbb{E}[q] - q^\*)| \le \gamma^n \beta.

So bias decays exponentially in n; variance saturates at \sigma^2 / (1 - \gamma^2) for large n. The MSE-optimal n is the n that balances the two — usually a small intermediate value when \beta is moderate and \sigma^2 is small.

P3. Show n-step Sarsa is Dvoretzky-shaped per state-action pair, with target T_k = G_t^{(n)}. Why is the contracting target condition non-trivially needed?

Hint 1 — a nudge The target depends on q for n steps in the future. The contraction has to play nicely with the n-step Bellman operator.
Full solution The expected n-step Bellman operator \mathcal{T}_\pi^n is a \gamma^n-contraction in \ell_\infty, with the same fixed point q_\pi. So Dvoretzky’s contracting-target condition holds with contraction modulus \gamma^n — even stronger contraction per “useful step” than n=1, but applied less often (only every n time steps). The trade is a wash in sample complexity but n-step is faster in episode count for long chains.

Practice — Stretch

S1. Why does the optimal n depend on the horizon?

Hint 1 — a nudge Reward propagation per episode versus variance per update.
Full solution Long horizon ⇒ more steps for reward to propagate ⇒ benefit of larger n (fewer episodes to push reward back). But long horizon also accumulates more reward variance ⇒ pressure for smaller n. The intersection of these two scaling laws picks the optimum, usually n \approx \sqrt{H} for horizon H on simple chains. Empirically Sutton-Barto’s bench observes n \in [3, 10] for horizons up to ~100.

S2. Sketch how to derive TD(λ) from n-step returns.

Hint 1 — a nudge Average all G_t^{(n)} with weight (1-\lambda)\lambda^{n-1}.
Full solution The λ-return is G_t^\lambda = (1 - \lambda) \sum_{n=1}^\infty \lambda^{n-1} G_t^{(n)}. The weights sum to 1 (geometric series). \lambda = 0 collapses to G_t^{(1)}; \lambda = 1 collapses to G_t^{(\infty)} = G_t. TD(λ) updates with this target. The naive implementation requires unbounded memory — but eligibility traces compute it incrementally in O(|\mathcal{S}\times\mathcal{A}|) memory per step. We don’t expand TD(λ) in this track; see Sutton-Barto Chapter 12.

Common confusions

  1. “n-step is biased because it bootstraps.” The bias exists but shrinks like \gamma^n in n. For \gamma = 0.99 and n = 10, the bias is 90% smaller than n=1.
  2. “Larger n is always better.” No — variance grows. The true winner is intermediate.
  3. “n-step needs to wait for the episode to end.” Wrong — only for n steps. That’s the point: it works in continuing tasks too.

Takeaways

If you remember nothing else:

  1. n-step return: G_t^{(n)} = \sum_{k=0}^{n-1} \gamma^k r_{t+k+1} + \gamma^n q(s_{t+n}, a_{t+n}).
  2. n is the bias-variance knob. Small n = low variance, biased; large n = high variance, unbiased; intermediate often best.
  3. Reward propagates n states per episode. Larger n means faster credit assignment on long chains.

Where this matters

AI / ML. Modern policy-gradient methods (A3C, PPO, IMPALA) use n-step returns as their value-function targets — typically n \in [5, 20] — because plain n=1 is too biased for deep-network estimators. The Generalised Advantage Estimator (GAE) is the \lambda-return version of the n-step idea, applied to advantage functions. See Bellman equation and learning rate.
