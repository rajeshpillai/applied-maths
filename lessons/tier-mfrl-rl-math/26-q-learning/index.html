26 · Q-Learning: TD for Optimal Action Values, Off-Policy

Replace the on-policy next action with the max — estimates q* regardless of the data-collection policy.

Pre-flight check

Before you start, you should be able to say yes to all four:

  1. Sarsa update from memory? — see MFRL — 24, Sarsa.
  2. Bellman optimality equation q^\*(s, a) = \mathbb{E}[r + \gamma \max_{a'} q^\*(s', a')]? — see MFRL — 10, Bellman Optimality.
  3. Difference between q_\pi and q^\* (one is for a fixed policy, the other is the best you can do)?
  4. ε-greedy policy? — see MFRL — 19, ε-Greedy.

If any answer is no, follow the link before continuing.

Touch

A 5×5 grid with a goal at the top-right and a “lava” cell that gives reward -10. Discount \gamma = 0.9.

The agent acts uniformly at random (every step picks a random direction, no greedy bias). Yet Q-learning’s q-table converges to q^\* — the optimal action values — because Q-learning’s target uses \max_{a'} q(s', a') regardless of which action the agent actually took.

Drag the slider on episodes; watch the heatmap of \max_a q(s, a) light up near the goal and dim near the lava.

viewof qEpisodes = Inputs.range([10, 1500], {value: 400, step: 10, label: "Episodes:"})
viewof qSeed = Inputs.range([1, 30], {value: 5, step: 1, label: "Seed:"})
{
  function mulberry32(a) {
    return function() {
      a |= 0; a = a + 0x6D2B79F5 | 0;
      let t = a;
      t = Math.imul(t ^ t >>> 15, t | 1);
      t ^= t + Math.imul(t ^ t >>> 7, t | 61);
      return ((t ^ t >>> 14) >>> 0) / 4294967296;
    };
  }
  const rand = mulberry32(qSeed * 23);
  const N = 5;
  const A = 4;     // 0=up, 1=down, 2=left, 3=right
  const gamma = 0.9, alpha = 0.2;
  const lava = {r: 2, c: 2};
  const goal = {r: 0, c: 4};
  const q = new Array(N * N * A).fill(0);
  const idx = (r, c, a) => (r * N + c) * A + a;
  function step(r, c, a) {
    let nr = r, nc = c;
    if (a === 0) nr = Math.max(0, r - 1);
    if (a === 1) nr = Math.min(N - 1, r + 1);
    if (a === 2) nc = Math.max(0, c - 1);
    if (a === 3) nc = Math.min(N - 1, c + 1);
    let reward = -0.04, done = false;
    if (nr === lava.r && nc === lava.c) { reward = -10; done = true; }
    if (nr === goal.r && nc === goal.c) { reward = 1; done = true; }
    return {nr, nc, reward, done};
  }
  for (let ep = 0; ep < qEpisodes; ep++) {
    let r = 0, c = 0;
    if (r === goal.r && c === goal.c) continue;
    for (let t = 0; t < 100; t++) {
      const a = Math.floor(rand() * A);   // FULLY RANDOM behaviour policy
      const {nr, nc, reward, done} = step(r, c, a);
      let maxQ = 0;
      if (!done) {
        maxQ = -Infinity;
        for (let ap = 0; ap < A; ap++) maxQ = Math.max(maxQ, q[idx(nr, nc, ap)]);
      }
      const target = reward + gamma * maxQ;
      q[idx(r, c, a)] += alpha * (target - q[idx(r, c, a)]);
      r = nr; c = nc;
      if (done) break;
    }
  }
  const cells = [];
  for (let r = 0; r < N; r++) for (let c = 0; c < N; c++) {
    let v = -Infinity;
    for (let a = 0; a < A; a++) v = Math.max(v, q[idx(r, c, a)]);
    cells.push({r, c, v});
  }
  const Plot = await import("https://cdn.jsdelivr.net/npm/@observablehq/plot@0.6/+esm");
  return Plot.plot({
    width: 360, height: 360,
    color: {scheme: "viridis", legend: true},
    x: {label: "col"}, y: {label: "row", reverse: true},
    marks: [
      Plot.cell(cells, {x: "c", y: "r", fill: "v"}),
      Plot.text(cells, {x: "c", y: "r", text: d => d.v.toFixed(1), fill: "white"})
    ]
  });
}

The agent acts randomly. Yet q tells the story of optimal play. That asymmetry — learn one policy while following another — is the core of off-policy learning.

Wonder

  1. Sarsa learns q_\pi for whatever \pi generated the data. Q-learning learns q^\* regardless of the data policy. Why does swapping q(s', a') for \max_{a'} q(s', a') buy that?
  2. The agent in the widget acts uniformly random — yet the heatmap shows the optimal policy. So can you learn from data collected by anybody — a human demonstrator, a bad policy, a random policy?
  3. Q-learning’s target is the Bellman optimality equation. Why does that matter? What does Sarsa’s target look like in comparison?
  4. If Q-learning learns q^\* from any data, why ever bother with on-policy methods?

Discover

Six problems to derive Q-learning, see the off-policy magic, and feel the catch.

Problem 1 — Bellman optimality recap

Write the Bellman optimality equation for q^\*(s, a).

Full solution q^\*(s, a) = \mathbb{E}\!\left[r + \gamma \max_{a'} q^\*(s', a') \,\bigg|\, s, a\right]. The expectation is over r, s' from the dynamics — there is no policy in sight on the right-hand side, because the optimal policy is “do the max.”

Problem 2 — the Q-learning target

Apply the Robbins-Monro recipe: a noisy sample of the right-hand side at (s, a) is…?

Full solution A noisy sample is r + \gamma \max_{a'} q(s', a'), where (r, s') is one observed transition. The bootstrap uses the current q in place of the unknown q^\*. Update: q(s, a) \leftarrow q(s, a) + \alpha [r + \gamma \max_{a'} q(s', a') - q(s, a)]. This is Q-learning.

Problem 3 — what’s a'?

In Sarsa the target uses q(s', a') where a' is the action actually taken. In Q-learning the target uses \max_{a'} q(s', a'). Why does this make Q-learning off-policy?

Hint 1 — a nudge Did the agent ever take the action that achieves the max?
Full solution The action a' in Q-learning’s target is the greedy action under the current q — but the agent does not need to take it. The agent could have acted ε-greedy, fully random, by demonstration — anything. The target is computed from q alone, not from the action taken. So the data-collection policy (called the behaviour policy) and the evaluated policy (the implicit greedy policy under q) are decoupled. That is the definition of off-policy.

Problem 4 — why this gives q^\* and not q_\pi

Sketch why Q-learning targets q^\* rather than q_\pi.

Hint 1 — a nudge What is the fixed point of the operator \mathbb{E}[r + \gamma \max_{a'} q(s', a') \mid s, a]?
Full solution The expected target is exactly the Bellman optimality operator \mathcal{T}^\*: (\mathcal{T}^\* q)(s, a) = \mathbb{E}[r + \gamma \max_{a'} q(s', a')]. Its unique fixed point is q^\*. So if Q-learning’s iterate converges, it converges to q^\* — independently of the policy generating data. The “max” inside the target is the BOE; that’s why Q-learning learns q^\*.

Problem 5 — convergence

State a sufficient condition for tabular Q-learning to converge to q^\*.

Full solution (Watkins-Dayan 1992): (i) every (s, a) visited infinitely often; (ii) per-(s,a) step sizes satisfy \sum \alpha_t = \infty, \sum \alpha_t^2 < \infty. Then q \to q^\* a.s. No GLIE requirement — because Q-learning’s target doesn’t care what the data-policy is, exploration just needs to cover.

Problem 6 — the maximisation bias

A subtle catch. With finite samples, \max_{a'} q(s', a') tends to overestimate the true max. Why?

Hint 1 — a nudge Jensen’s inequality: \mathbb{E}[\max] \ge \max \mathbb{E}.
Full solution \mathbb{E}[\max_a q(s', a)] \ge \max_a \mathbb{E}[q(s', a)] = \max_a q^\*(s', a). Because we max over noisy estimates, we systematically overshoot — the maximisation bias. Double Q-learning (Hasselt 2010) fixes this by maintaining two q-tables, one to select the argmax action and the other to evaluate it. We won’t expand here; see Hasselt’s paper.

The Sarsa-vs-Q-learning difference is one symbol: Sarsa uses q(s', a') (the action the policy chose); Q-learning uses \max_{a'} q(s', a') (the action the optimal policy would choose). That single substitution swaps on-policy for off-policy and q_\pi for q^\*.

Christopher Watkins introduced Q-learning in his 1989 Cambridge PhD thesis. He proved convergence in the same thesis — the first proof of its kind. The 2013 DeepMind paper that ignited deep RL used Q-learning with a deep neural network and called the result Deep Q-Network (DQN). The 2015 Nature paper showing DQN reaching human-level Atari from raw pixels is, at its core, Watkins’s 1989 update with two engineering tricks — target networks and replay buffers (covered in Lesson 31).

Name

Q-learning — off-policy TD(0) for optimal action values. Given a sample tuple (s_t, a_t, r_{t+1}, s_{t+1}) collected under any behaviour policy b, update q(s_t, a_t) \;\leftarrow\; q(s_t, a_t) + \alpha_t \, [\,\underbrace{r_{t+1} + \gamma \max_{a'} q(s_{t+1}, a')}_{\text{Q-learning target}} - q(s_t, a_t)\,]. Under (i) every (s, a) visited infinitely often and (ii) RM step conditions, q \to q^\* a.s. No constraint on b except coverage.

Off-policy = the data-collection (behaviour) policy may differ from the policy being evaluated/improved (target).

A short verse, in the spirit of Bhāskara’s Līlāvatī:

Reward I just saw, plus the best of the next — Not the action I took, but the action I would. Behaviour and target part company, And I learn the best policy from any data at all.

Apply

Worked example — Q-learning step on a tiny grid

States s_1, s_2. From s_1 actions {right} and {left}. Right goes to s_2 (terminal, reward +1); left self-loops (reward -1). \gamma = 0.9, \alpha = 0.5. Initial q = 0. Suppose the behaviour policy chose left, giving the tuple (s_1, \text{left}, r=-1, s_1). Compute the Q-learning update.

A competent solver thinks like this:

  1. Target: r + \gamma \max_{a'} q(s_1, a') = -1 + 0.9 \cdot \max(q(s_1, \text{left}), q(s_1, \text{right})) = -1 + 0.9 \cdot 0 = -1.
  2. Update q(s_1, \text{left}): 0 + 0.5 \cdot (-1 - 0) = -0.5.
  3. Suppose next the behaviour policy also chooses left: tuple (s_1, \text{left}, r=-1, s_1) again.
  4. Target: -1 + 0.9 \cdot \max(-0.5, 0) = -1 + 0 = -1. Update q(s_1, \text{left}) = -0.5 + 0.5 \cdot (-0.5) = -0.75.
  5. Eventually random behaviour selects right and q(s_1, \text{right}) gets updated. Even from purely left-biased data Q-learning would eventually learn that right is better — as long as right is sampled sometimes (the coverage condition).

Practice — Warm

W1. What is the Q-learning target if r = 2, \gamma = 0.95, q(s', \text{up}) = 4, q(s', \text{down}) = 7, q(s', \text{stay}) = 5?

Full solution \max_{a'} q(s', a') = 7. Target = 2 + 0.95 \cdot 7 = 8.65.

W2. True or false: “Q-learning needs ε-greedy behaviour to converge.”

Full solution False — only coverage is needed. Q-learning converges to q^\* even under uniformly random or expert-demonstration behaviour, as long as every (s, a) is visited infinitely often. The target doesn’t care.

W3. Spot the difference: in Sarsa we required GLIE for q \to q^\*; in Q-learning we don’t. Why?

Full solution Sarsa learns q_\pi for the behaviour policy \pi. So you need \pi \to \pi^\* (the GLIE part) to get q_\pi \to q^\*. Q-learning learns q^\* directly via the BOE; the behaviour just supplies coverage.

Practice — Working

P1. Show that Q-learning is Dvoretzky-shaped per (s, a). What plays the role of “contracting target”?

Hint 1 — a nudge The expected target is the Bellman optimality operator.
Full solution q(s,a)_{k+1} = (1 - \alpha) q(s,a)_k + \alpha [r + \gamma \max_{a'} q(s', a')_k]. The expectation of the bracket is (\mathcal{T}^\* q_k)(s, a), where \mathcal{T}^\* is the BOE-operator and a \gamma-contraction in \ell_\infty (lesson 11). So target is contracting toward q^\*; Dvoretzky’s theorem applies; q \to q^\* a.s.

P2. Sarsa under uniform-random behaviour: what does it converge to, and why is that not q^\*?

Hint 1 — a nudge What policy generated the a' in Sarsa’s target?
Full solution Sarsa’s target uses the actually-taken a' — uniform-random in this case. So Sarsa learns q_\pi where \pi is uniform random — the value of acting randomly. That’s nowhere near q^\* unless the MDP is degenerate. Lesson: Sarsa’s target inherits the behaviour policy; Q-learning’s does not.

P3. A continuous-action problem (e.g., robot torque). Why does Q-learning’s \max_{a'} become a practical obstacle?

Full solution For continuous a, computing \max_{a'} q(s', a') requires solving a continuous optimisation at every step. That is hopeless for arbitrary q and motivates actor-critic methods (Lessons 35-38), where a separate parameterised actor proposes the argmax action and is itself trained by gradient ascent on q.

Practice — Stretch

S1. The deadly triad preview. Name the three ingredients that, when combined, can cause Q-learning with function approximation to diverge.

Full solution Bootstrapping (TD-style targets), off-policy learning, and function approximation (parameterised q, especially neural nets). Tabular Q-learning has the first two but not the third — and is safe. Adding FA breaks the safety; this is Baird’s counterexample, expanded in Lesson 30. Modern deep-RL papers use target networks and replay buffers to stabilise.

S2. Maximisation bias quantified. Suppose we have K noisy estimators \hat q_1, \ldots, \hat q_K of q, each unbiased with variance \sigma^2. Show \mathbb{E}[\max_k \hat q_k] exceeds \max_k q_k by at least \sigma \sqrt{\log K} (very roughly).

Hint 1 — a nudge Order statistics of K Gaussians.
Full solution For i.i.d. \mathcal{N}(\mu, \sigma^2), \mathbb{E}[\max] \approx \mu + \sigma \sqrt{2 \log K} (extreme-value theory). Even if each estimator is unbiased individually, their max is biased upward by \Theta(\sigma \sqrt{\log K}). Q-learning systematically over-estimates the value of the best action, especially in high-action settings. Double Q-learning fixes this.

Common confusions

  1. “Q-learning is on-policy because the agent uses ε-greedy.” No — acting ε-greedy doesn’t make the algorithm on-policy. What matters is what’s in the target. Q-learning’s target uses \max, not a', so it’s off-policy regardless of how you act.
  2. “Sarsa and Q-learning differ only by the data tuple.” Almost — Sarsa needs a', Q-learning doesn’t. But the more important difference is the fixed point they target (q_\pi vs q^\*).
  3. “Off-policy means we don’t need exploration.” No. We need coverage — every (s, a) must be visited. Without coverage, you can’t update some q entries and the algorithm can’t reach q^\*.

Takeaways

If you remember nothing else:

  1. Q-learning update: q(s, a) \leftarrow q(s, a) + \alpha [r + \gamma \max_{a'} q(s', a') - q(s, a)]. The “max” is the BOE.
  2. Off-policy. Behaviour policy and target policy can differ; only coverage is required.
  3. Targets q^\* directly. No GLIE needed — but watch out for maximisation bias and the deadly triad with FA.
